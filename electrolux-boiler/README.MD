# Electrolux Boiler (ESPHome + ESP32)

ESPHome configuration and 3D-printable enclosure parts for integrating an Electrolux boiler with Home Assistant (via MQTT / API) using an ESP32 and a UART connection.

> рҹ’Ў **Standalone monitoring without grid power:** The ESP32 can be powered independently via its **5 V USB pin** (as shown in the wiring photos). When the boiler has no mains power (grid outage, circuit breaker off, etc.), the boiler's internal controller вҖ” including its relay, thermostat, and temperature sensor вҖ” continues to operate on its own low-voltage supply. The ESP32 will still receive UART status packets and can report **current water temperature** to Home Assistant even with no grid power to the heating element.

### Timer Mode

The timer is a **scheduled start time** (not a countdown). When activated:
1. Boiler clock is synced to NTP (`Sync Boiler Clock`)
2. Timer packet is sent with the configured `HH:MM` start time
3. State is requested immediately вҖ” boiler should report mode = `Timer` (4) to confirm

The boiler will start heating at the scheduled clock time using the current target temperature.

> вҡ пёҸ **Timer mode is treated as not-heating**: while the boiler reports `Timer` mode it is idle (waiting for the scheduled start), so `is_heating` is always `false`. This means no energy is accumulated and the temperature trend is clamped to `Stable` or `Down`.

### Energy Tracking

- **Boiler Power** reflects the element's rated wattage (700 / 1300 / 2000 W) while `is_heating` is true
- **Boiler Total Energy** accumulates Wh every 30 s and survives reboots (`restore_value: true`)
- Energy is **not** accumulated in Timer mode вҖ” the boiler is idle while waiting for the scheduled start time
- Add **Boiler Total Energy** to the HA Energy Dashboard under *Settings вҶ’ Energy вҶ’ Individual devices*

### Temperature Trend

The `Boiler Temp Trend` sensor publishes `Up`, `Down`, or `Stable` using a **least-squares linear regression** over a 4-sample ring buffer (one sample per ~30 s packet):

- **Up threshold** is mode-dependent вҖ” faster heating modes require a steeper slope to avoid false positives:

  | Mode | Up slope threshold |
  |------|--------------------|
  | 2000W (High) | 0.50 В°C/sample |
  | 1300W (Medium) | 0.35 В°C/sample |
  | 700W (Low) / Timer | 0.20 В°C/sample |
  | No Frost / Off | 0.10 В°C/sample |

- **Down threshold** is fixed at 0.10 В°C/sample вҖ” cooling is always passive and gradual
- When `is_heating` is `false`, the result is clamped to `Stable` or `Down` вҖ” temperature cannot rise without active heating

### Protocol Reference

- All packets: `AA [len] [dataвҖҰ] [CS]` вҖ” CS = sum of all preceding bytes (uint8)
- TX write command `0x0A`: sub-commands `0`=mode, `1`=clock, `2`=timer, `3`=BST
- RX status packet `0x09` / `0x88`: 13 bytes, 30 s cadence
- Reference: [dentra/esphome-ewh вҖ” reverse.md](https://github.com/dentra/esphome-ewh/blob/master/reverse.md)

## Quick Setup

1. **Copy secrets** вҖ“ create / extend `secrets.yaml`:
   ```yaml
   mqtt_broker: <your_broker_ip>
   mqtt_username: <username>
   mqtt_password: <password>
   wifi_ssid: <your_ssid>
   wifi_password: <your_wifi_password>
   ```
2. **Flash** вҖ“ compile and flash `electrolux-boiler.yaml` to the ESP32.
3. **Wire** вҖ“ connect UART as described above (add level shifting on the RX line).
4. **Home Assistant** вҖ“ entities appear automatically via MQTT discovery.
5. **Energy Dashboard** вҖ“ add **Boiler Total Energy** sensor under *Settings вҶ’ Energy вҶ’ Individual devices*.
6. **Fallback** вҖ“ if WiFi is unavailable, connect to AP `AP-Boiler` (password: `password`) and configure via captive portal or web UI at `192.168.4.1`.
